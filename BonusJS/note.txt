


Three (3) Main Reasons to Use Each Method

1️⃣ element.textContent = "Hello World";

Main Reasons to Use:
Safe (prevents HTML injection)
Fast
Shows exactly the text

Limitation:
element.textContent = "<b>Hello</b>";
// Output: <b>Hello</b> (not bold)

2️⃣ element.innerText = "Hello World";

Main Reasons to Use:
Only reads visible text (ignores hidden with CSS)
Respects CSS styling
Useful when you care about what the user actually sees

Limitation:
Slower than textContent
Rarely needed unless visibility matters

3️⃣ element.innerHTML = "<b>Hello</b>";

Main Reasons to Use:
Can insert HTML tags
Can create elements dynamically
Useful for rich content

Limitation / Risk:
element.innerHTML = userInput; 
// ❌ Dangerous: can cause XSS if input is from a user
